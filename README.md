# Private Chat — закрытый PWA-мессенджер на Cloudflare Workers

Готовый стартовый проект для узкого круга пользователей:
- вход и регистрация только по инвайт-коду;
- личные текстовые чаты один на один;
- realtime через WebSocket + Durable Objects;
- история сообщений в D1;
- PWA для iPhone;
- Web Push для установленного PWA;
- админ может создавать новые инвайты.

## Что внутри
- `src/index.js` — API, авторизация, работа с D1, пуши;
- `src/chat-room.js` — Durable Object для realtime-соединений;
- `public/` — фронтенд PWA;
- `schema.sql` — схема базы D1;
- `wrangler.jsonc` — конфиг Worker;
- `scripts/generate-vapid.mjs` — генерация VAPID-ключей для push.

## Что нужно сделать вам

### 1) Загрузить проект в GitHub
Создайте новый репозиторий и загрузите туда все файлы проекта.

### 2) Установить зависимости локально
Откройте терминал в папке проекта:

```bash
npm install
```

### 3) Авторизоваться в Cloudflare
```bash
npx wrangler login
```

### 4) Создать D1-базу
```bash
npx wrangler d1 create private_messenger
```

Wrangler вернёт `database_id`.
Вставьте его в `wrangler.jsonc` вместо:

```json
PUT_YOUR_D1_DATABASE_ID_HERE
```

### 5) Применить схему базы
```bash
npx wrangler d1 execute private_messenger --remote --file=./schema.sql
```

### 6) Сгенерировать VAPID-ключи
```bash
npm run gen:vapid
```

Скрипт выведет:
- `VAPID_PUBLIC_KEY`
- `VAPID_PRIVATE_KEY_PEM`
- `VAPID_SUBJECT`

### 7) Задать секреты Worker
Задайте секреты через Wrangler:

```bash
npx wrangler secret put BOOTSTRAP_SECRET
npx wrangler secret put VAPID_PRIVATE_KEY_PEM
npx wrangler secret put VAPID_PUBLIC_KEY
npx wrangler secret put VAPID_SUBJECT
npx wrangler secret put ADMIN_INVITE_CODE
```

Рекомендации:
- `BOOTSTRAP_SECRET` — любая длинная случайная строка, только для первичного создания первого инвайта;
- `ADMIN_INVITE_CODE` — можете задать сами удобный код, например `AYLA-ADMIN-2026`, либо оставить пустым и использовать случайный из bootstrap.

### 8) Локально проверить
```bash
npm run dev
```

### 9) Задеплоить
```bash
npm run deploy
```

После деплоя получите URL Worker.

## Первичный запуск

### Создать самый первый инвайт
После деплоя выполните запрос:

```bash
curl -X POST "https://YOUR-WORKER-URL/api/bootstrap" \
  -H "content-type: application/json" \
  --data '{"secret":"ВАШ_BOOTSTRAP_SECRET"}'
```

В ответе придёт `inviteCode`.

### Зарегистрировать первого пользователя
Откройте сайт Worker на iPhone или в браузере, нажмите «Регистрация» и введите:
- инвайт-код;
- логин;
- имя;
- пароль.

Первый пользователь автоматически становится `admin`.

### Создавать новые инвайты
После входа слева откроется админ-панель, где можно создавать новые инвайты для своих людей.

## Как установить как PWA на iPhone
1. Откройте адрес сайта в Safari.
2. Нажмите «Поделиться».
3. Выберите «На экран Домой».
4. Запустите приложение уже с иконки.
5. Внутри чата нажмите «Включить уведомления».

## Что уже умеет
- закрытая регистрация по инвайтам;
- хранение пользователей и сессий;
- личные чаты;
- мгновенная доставка новых сообщений в открытом чате;
- пуш-уведомления без внешнего мессенджера;
- админ-инвайты.

## Чего пока нет
- звонков;
- голосовых сообщений;
- групповых чатов;
- сквозного end-to-end шифрования;
- удаления сообщений;
- вложений и фото;
- мультидевайс-синхронизации продвинутого уровня;
- тонкой админки для блокировки устройств.

## Важная правда
Это хорошая **v1-версия приватного закрытого чата для своих**, но это **не замена Signal по уровню криптографической зрелости**.

Для вашего сценария это нормально, потому что задача здесь — сделать:
- независимый канал;
- без Telegram / WhatsApp / MAX;
- для малого круга;
- быстро и управляемо.

## Что я бы делал следующим этапом
После запуска v1 логично добавить:
1. группы;
2. удаление/редактирование сообщений;
3. вложения через R2;
4. блокировку/деактивацию пользователей;
5. отдельный экран сессий;
6. более сильную криптографию поверх transport security.
